# UI / UX 規格

## 色彩系統

| 用途 | 色碼 | 說明 |
|------|------|------|
| 主色 | `#1a56db` | Header、邊框、主按鈕 |
| 強調 | `#fbbf24` | Badge、件數標籤 |
| 成功 | `#059669` | 新增成功、匯出按鈕 |
| 危險 | `#ef4444` | 清除、刪除按鈕 |
| 背景 | `#f0f2f5` | 頁面底色 |
| 卡片 | `#ffffff` | 功能區塊背景 |
| 次要文字 | `#6b7280` | 說明文字、欄位標籤 |

## 字型
- 系統字型：`Arial, sans-serif`
- 條碼、品號等等寬資料：`monospace`

---

## 版面結構（上至下）

```
[Header] 標題 + 筆數 Badge（sticky）
[Card] 掃描輸入區        ← 最重要，視覺最大
[Card] 品號 + 數量設定
[Card] 掃描記錄列表      ← 可捲動
[Card] 品號統計摘要
[Card] 操作按鈕群
[StatusBar] 說明文字
```

## 響應式
- 單欄，最大寬度 900px，置中
- 所有元素在 320px 寬的 Android PDA 上必須正常顯示
- 按鈕高度 ≥ 44px（手指可點）

---

## 掃描輸入框規格

- 字型大小：`1.8rem`，字體：`monospace`
- 邊框：3px solid `#1a56db`
- 背景：`#eff6ff`（預設）→ `#dbeafe`（focus）
- 整寬（100%）
- placeholder：`請掃描或輸入條碼…`

### 焦點管理規則（重要）
1. 頁面載入時焦點在掃描框
2. 使用者點擊任何非輸入欄位 → 焦點自動回到掃描框
3. 刪除記錄後 → 焦點回到掃描框
4. 清除全部後 → 焦點回到掃描框
5. 品號輸入框和數量輸入框**例外**（允許使用者在這兩個欄位停留）

---

## 新增動畫

- 新記錄加入時：該列觸發 `flash` 動畫（0.6s，從綠色 `#bbf7d0` 漸褪）
- 最新掃描提示：顯示 `✓ 條碼 ／ 品號 × 數量`，2秒後淡出至 50% 透明度

---

## 按鈕規格

| 按鈕 | 背景 | 用途 |
|------|------|------|
| 下載 Excel | `#059669` 白字 | 主要匯出 |
| 複製（貼到 Excel）| `#1a56db` 白字 | 備用匯出 |
| 清除全部記錄 | `#ef4444` 白字 | 危險操作，需 confirm 確認 |

- 點擊時 opacity 降至 0.75（feedback）
- border-radius：8px

---

## 記錄列表規格

- 倒序顯示（最新在上）
- 欄位：# ／ 時間 ／ 條碼 ／ 品號 ／ 數量 ／ 刪除
- 最大高度 340px，超出捲動
- 空列表顯示：`尚無記錄`（灰色置中）
- 刪除按鈕：✕ 符號，紅色，hover 效果

---

## 統計表規格

- 欄位：品號 ／ 筆數 ／ 總數量
- 總數量靠右對齊、粗體
- 合計列：粗體 + 淺灰背景 `#f9fafb`

---

## 狀態列

- 底部固定說明文字：`資料儲存於本機瀏覽器，重新整理不會遺失`
- 字型：0.8rem，灰色 `#6b7280`

---

## v2-cf-pages 額外 UI 規則

- 登入/裝置識別：頁首右側顯示「裝置名稱」（簡單文字，不需帳號系統）
- 同步狀態：Badge 旁顯示 `● 已同步` / `○ 同步中` / `✕ 離線`
- 照片預覽：記錄列表每列可展開顯示縮圖（點擊放大）
- 多人警告：同一條碼被其他裝置掃描時，顯示黃色提示
